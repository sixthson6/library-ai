# Document Ingestion Implementation Plan

## Goal Overview
We will implement an ingestion pipeline to receive documents, break them down, generate embeddings, and store them securely in your PostgreSQL database using the `pgvector` extension.

## Proposed Steps & Exact Order

### 1. Verify Database Setup (pgvector)
Before accepting any files, the application will ensure the database is ready:
- Connect to the PostgreSQL database on startup.
- Execute a check (e.g., `SELECT * FROM pg_extension WHERE extname = 'vector';`) or run `CREATE EXTENSION IF NOT EXISTS vector;` to guarantee `pgvector` is installed.
- Establish the database schema/table (e.g., a `document_chunks` table containing columns for `id`, `filename`, `chunk_text`, and `embedding` defined as a `VECTOR` type).

### 2. Define the Upload Endpoint
- In `routers/documents.py`, create a `POST /documents/upload/` endpoint.
- It will accept an `UploadFile` parameter to allow users to upload text, PDF, or other supported document formats.

### 3. Document Processing Pipeline (In Order)
When a document is uploaded, it will go through the following synchronous or asynchronous pipeline:
1. **Extraction**: Read the uploaded file's binary stream into memory and extract the raw text content.
2. **Chunking**: Pass the raw text through a text splitter (e.g., chunking by character count, paragraphs, or semantic boundaries). This creates smaller, contextually rich segments of text.
3. **Embedding Generation**: Pass each text chunk to an embedding model (e.g., an OpenAI embedding model or a local HuggingFace model via `sentence-transformers`). This converts the human-readable text into mathematical vectors (arrays of floats).
4. **Metadata Association**: Tie each resulting vector back to its source text chunk, filename, and chunk index to ensure traceability.

### 4. Vector Database Storage
- Connect to the PostgreSQL database using an ORM like SQLAlchemy (with `pgvector-python`) or an async driver like `asyncpg`.
- Bulk insert the structured records (filename, chunk text, and the vector embedding) into your target vector table.
- Return a success response to the user with ingestion statistics (e.g., "Successfully ingested document into 15 chunks").
