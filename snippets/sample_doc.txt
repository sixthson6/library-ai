# Introduction to Vector Databases

A vector database is a type of database designed to store, manage, and query high-dimensional vectors efficiently. Unlike traditional relational databases that query based on exact matches or keyword proximity, vector databases use mathematical distance metrics to find data that is semantically similar.

## How it works

When you want to store a document, you first pass it through an embedding model (like those from OpenAI or HuggingFace). This model converts the textual content into an array of floating-point numbers known as a "vector embedding." This array captures the semantic meaning of the text.

Once stored, you can query the database by converting your search query into a vector using the exact same model. The database then calculates the "distance" (such as cosine similarity or Euclidean distance) between your query vector and all the stored vectors. The results with the shortest distance are the most contextually relevant, even if they don't share exact keywords with your search query.

## Use Cases

1. **Retrieval-Augmented Generation (RAG)**: Providing context to Large Language Models (LLMs) so they can answer questions based on your specific private data.
2. **Semantic Search**: Searching through internal documents, wikis, or product catalogs where users might use synonyms or descriptive phrases.
3. **Recommendation Systems**: Finding products or content that are "similar" to what a user is currently viewing.

By chunking large documents into smaller pieces before generating embeddings, we ensure that the specific context returned is highly focused and fits neatly within an LLM's context window.
